    Programação Orientada a Objetos em Python¶

    Introdução¶

    Programação Procedural¶

    Um paradigma de programação entre muitos outros.

    O programa é dividido é partes menores chamadas métodos. Os métodos são entidades básicas usadas para construir um programa.

    Tem principal vantagem o reaproveitamento de código. Porém, é difícil implementar em casos reais complexos.

    Programação Orientada a Objeto - OOP¶

    É um paradigma de programação que se vale do conceito de classes e objetos.

    As entidade básicas na programação orientada a objetos são classes e objetos. A ideia básica da OOP é dividir um programa complexo em uma quantidade de objetos que se comunicam entre si.

    Os objetos de um programa frequentemente representam objetos do mundo real. Mas também podem implementar a lógica da aplicação sem qualquer analogia direta com o mundo real.

    Objetos e Classes¶

    Objetos podem conter dados na forma de variáveis e métodos que operam nestes dados. Objetos são uma coleção de dados e seus comportamentos.

    Um objeto tem um estado e um comportamento.

    Os objetos são criados pelas classes, que funcionam como um molde para a criação de objetos.

    Classes são estruturas de dados definidas pelo usuário.

    Classes e Objetos em Python¶

    Propriedades/Atributos: São variáveis que contém informação sobre um objeto ou classe.

    Métodos: Métodos são como funções que têm acesso a propriedades e outros métodos de uma classe. Métodos podem aceitar parâmetros e retornar valores. São usados para executar uma ação em um objeto de uma classe.

    • Objetos e classes nos permitem criar aplicações complexas em python.

    • Objetos e classes contribuem para a compartimentalização do código. Diferentes componentes podem ser tornar classes separadas que interagem através de interfaces, esses componentes prontos podem se tornar disponíveis para uso em aplicações futuras.

    • O uso de classes torna mais fácil de realizar manutenção em diferentes partes de uma aplicação, pois é mais fácil fazer mudanças em classes.

    Criando uma Classe¶

    Declaração¶

    A keyword class diz ao compilador que estamos criando uma classe.

    class ClassName:
    pass

    Regras de Nomeação¶

    1. Deve começar com uma letra ou um _.
    2. Deve conter apenas letras, números e _.

    Criando um Objeto de uma Classe¶

    class MyClass:
    pass

obj = MyClass()

    Implementando Propriedades¶

    Class Servidor:
    nome = "Steve"
    matricula = 43199951
    tempo = 1

    Acessando Propriedades e Atribuindo Valores¶

    steve = Servidor()

print(steve.matricula)
>>> 43199951

steve.matricula = 43188877
print(steve.matricula)
>>> 43188877

    Atributos podem ser criados fora de uma classe¶

    steve = Servidor()
steve.cargo = "Chefe"
print(steve.cargo)

    Inicializando Objetos¶

    É um usado um inicializador para inicializar objetos de uma classe. É um método especial que delineia os passos que serão executados quando um objeto de uma classe é criado em um programa.

    O método de inicialização é tem um nome pré definido em python __init__.

    Os underlines duplos significam que este é um método especial que o interpretador de Python irá tratar como um caso especial.

    O inicializar é um método especial porque não tem um tipo de retorno. O primeiro parâmetro de __init__ é self, que é uma forma de se referir ao objeto que está sendo inicializado.

    Definindo inicializadores¶

    class Servidor:
    def __init__(self, nome, matricula, tempo):
        self.nome = nome
        self.matricula = matricula
        self.tempo = tempo

Steve = Servidor("Steve", 43199951, 3)

    O inicializador é chamado automaticamente quando um objeto de uma classe é criado, ele atribui valores às variáveis da instância.

    Variáveis da Classe e da Instância¶

    As variáveis da classe são divididas por todas as instâncias (ou objetos) da classe. Uma mudança na variável da classe irá mudar o valor da propriedade em todos os objetos da classe.

    As variáveis da instância são únicas para cada objeto da classe. Uma mudança na variável da instância irá afetar apenas o valor da propriedade deste objeto específico.

    Variáveis de Classe são inicializadas fora do inicializador e as variáveis da instância são definidas dentro do inicializador.

    Implementando Métodos em uma classe¶

    Existem 3 tipos de métodos em Python:

    1. Métodos da instância.
    2. Métodos da classe.
    3. Métodos estáticos.

    Métodos da instância são os métodos mais comumente utilizados em python.

    Os métodos tem como propósito alterar o conteúdo das propriedades ou usar seus valores para executar alguma operação.

    Definição e Declaração¶

    Um método é um grupo de declarações que executa algumas operações e pode ou não retornar um resultado.

    Parâmetros do Método¶

    Podem ser passados valores para um método através dos parâmetros do método. Em Python, o primeiro parâmetro da um método é sempre self.

    O argumento self¶

    É um pseudo-variável que se refere ao objeto sendo chamado (o objeto ao qual o método ou propriedade pertence). Se o usuário não mencionar self como primeiro argumento, o primeiro argumento será tratado como referência ao objeto, self é uma convenção.

    self só precisa ser passado como argumento na definição do método e não quando o método é chamado.

    Métodos da Classe¶

    Métodos da classe trabalham do variáveis da classe e são acessíveis utilizando o nome da classe ao invés do objeto. Como todos os objetos da classe compartilham as variáveis de classe, métodos da classe são usados para acessar e modificar variáveis da classe.

    Para declarar um método como método de classe utilizamos o decorator @classmethod. Assim como self é usado para objetos, cls é utilizado para se referir à classe.

    class Player:
    teamName = 'Liverpool'  # class variables

    def __init__(self, name):
        self.name = name  # creating instance variables

    @classmethod
    def getTeamName(cls):
        return cls.teamName

print(Player.getTeamName())

    Métodos Estáticos¶

    São métodos usualmente limitados apenas à classe. Eles não têm relação direta com as variáveis da classe ou das instâncias. São usados como funções de utilidade dentro de uma classe, ou quando não queremos classes herdades para modificar a definição de um método.

    Métodos estáticos podem ser acessados utilizando o nome da classe ou do objeto.

    Para declarar um método como método de classe utilizamos o decorator @staticmethod. Assim como self é usado para objetos, cls é utilizado para se referir à classe.

    class Player:
    teamName = 'Liverpool'  # class variables

    def __init__(self, name):
        self.name = name  # creating instance variables

    @staticmethod
    def demo():
        print("I am a static method.")

p1 = Player('lol')
p1.demo()
Player.demo()

    Modificadores de Acesso¶

    Atributos públicos¶

    São atributos que podem ser acessados dentro e fora da classe.

    Tecnicamente, todos os métodos e atributos são publicamente disponíveis por padrão. Se quisermos sugerir que um método não deve ser usado publicamente, temos que declarar explicitamente como privado.

    Atributos privados¶

    Não podem ser acessados diretamente de fora de classe, mas podem ser acessados dentro da classe.

    O objetivo é manter eles escondidos de usuários e outras classes. Podemos tornar membros privados utilizado o prefixo __. Tentar acessar um atributo privado irá gerar um erro.

    Métodos Privados¶

    Usar o prefixo __.

    Acessando atributos privados no código principal¶

    Métodos e Propriedades com __ são feitas para assegurar que o usuário não acesse os de forma descuidada. Porém, se o acesso for necessário, eles ainda podem ser acessado utilizando a sintaxe:

    _<Classe>__<propriedade>.

    Ocultação de Informações¶

    Uma boa prática para tornar o sistema de orientação a objeto mais confiável e livre de erros, é limitar o acesso aos membros da classes.

    Ocultação de Informações se refere ao conceito de esconder o funcionamento interno de uma classe e simplesmente fornecer uma interface pela qual o mundo externo pode interagir com a classe sem saber o que acontece por dentro.

    Os Componentes da Ocultação de Informações são Encapsulamento e Abstração.

    Encapsulamento¶

    É uma técnica fundamental em OOP. Se refere à vinculação de dados e métodos para manipular dados em uma única unidade, uma classe.

    Assim, uma classe pode ser vista como uma capsula contendo dois lados, métodos e variáveis.

    Ao encapsular classes, uma convenção é declarar todas as variáveis de uma classe como privado, isto irá restringir o acesso direto por código fora da classe. Para acessar estes métodos são implementados métodos públicos para o mundo externo se comunicar com a classe. Estes métodos são chamados de getters e setters.

    Vantagens do encapsulamento¶

    • Classes tornam o código fácil de mudar e manter.
    • Propriedades a ser escondidas podem ser especificadas facilmente.
    • Podemos decidir quais classes externas ou funções podem acessar as propriedades da classe.

    Getters e Setters¶

    Um método getter permite a leitura do valor de uma propriedade.

    Um método setter permite a modificação do valor de uma propriedade.

    Uma convenção comum é utilizar a nomeação getProperty() e setProperty() para estes métodos.

    class User:
    def __init__(self, username=None):  # defining initializer
        self.__username = username
    def setUsername(self, x):
        self.__username = x
    def getUsername(self):
        return (self.__username)

Steve = User('steve1')
Steve.setUsername('steve2')

    Herança¶

    Herança é outro conceito chave em programação orientada a objetos.

    Herança fornece uma forma de criar uma nova classe a partir de uma classe existente. A nova classe é uma versão especializada da classe existente, herdando todos as variáveis não privadas e métodos da classe existente. A classe existente é usada como base para criar a nova classe.

    Quando criamos uma classe em Python, estamos criando uma classe filha do objeto class do Python, o que já é um exemplo de Herança.

    Terminologia¶

    Classe pai / Super Classe / Classe Base: Esta classe permite a reutilização das suas propriedades públicas em outra classe;

    Classe Filha / Sub Classe / Classe Derivada: Classe que herda ou estende a superclasse. A classe filha tem todos os atributos públicos da classe pai.

    Sintaxe Python¶

    class ParentClass:
    pass
class ChildClass(ParentClass):
    pass

    Exemplo:

    class Vehicle:
    def __init__(self, make, color, model):
        self.make = make
        self.color = color
        self.model = model

    def printDetails(self):
        print("Manufacturer:", self.make)
        print("Color:", self.color)
        print("Model:", self.model)


class Car(Vehicle):
    def __init__(self, make, color, model, doors):
        Vehicle.__init__(self, make, color, model)
        self.doors = doors

    def printCarDetails(self):
        self.printDetails()
        print("Doors:", self.doors)

obj1 = Car("Suzuki", "Grey", "2015", 4)
obj1.printCarDetails()

    A função Super()¶

    A função super() é usada na classe filha para se referir à classe pai sem precisar nomear explicitamente. Serve para deixar o código mais fácil de administrar e elimina a necessidade de saber o nome da classe pai para acessar seus atributos.

    Pode ser usada para chamar atributos, métodos e inicializar a classe.

    Propriedade:

    class Vehicle: 
    fuelCap = 90

class Car(Vehicle): 
    fuelCap = 50
    def display(self):
        print("Fuel cap from the Vehicle Class:", super().fuelCap)
        print("Fuel cap from the Car Class:", self.fuelCap)

    Método:

    class Vehicle:
    def display(self):
        print("I am from the Vehicle Class")

class Car(Vehicle):
    def display(self):
        super().display()
        print("I am from the Car Class")

    Tipos de Herança¶

    Herança Simples¶

    Existe apenas uma única classe estendendo de outra classe.

    flowchart BT
b[classe filha] --> a[classe pai]

    Herança Multinível¶

    Quando uma classe é derivada de outra classe que também é derivada de outra classe.

    flowchart BT
a[classe filha] --> b[classe pai] --> c[classe avô]

    Herança Hierárquica¶

    Quando múltiplas classes são derivadas de uma mesma classe pai.

    flowchart BT
a[classe filha01]  --> c[classe pai]
b[classe filha02] --> c[classe pai]

    Herança Múltipla¶

    Quando a classe filha tem mais de uma classe pai imediata.

    flowchart BT
a[classe filha01]  --> b[classe pai01]
a --> c[classe pai02]

    Herança Híbrida¶

    Combinação de Herança Múltipla e Herança Multinível.

    flowchart BT
a[classe filha01]  --> b[classe pai01] --> d[classe avô]
a --> c[classe pai02] --> d

    Sintaxe Para Herança Múltipla¶

    As duas ou mais classes pai são passadas como parâmetro para a criação da classe filha.

    class CombustionEngine():  
    def setTankCapacity(self, tankCapacity):
        self.tankCapacity = tankCapacity

class ElectricEngine():  
    def setChargeCapacity(self, chargeCapacity):
        self.chargeCapacity = chargeCapacity

class HybridEngine(CombustionEngine, ElectricEngine):
    def printDetails(self):
        print("Tank Capacity:", self.tankCapacity)
        print("Charge Capacity:", self.chargeCapacity)

    Vantagens da Herança¶

    • Reusabilidade.
    • Modificação de código: É mais fácil mudar o código de uma classe pai do que mudar o código em várias classes filhas.
    • Extensibilidade: O código da classe base pode ser estendido de acordo com as necessidades das classes derivadas, tornando fácil de atualizar o código e melhorar partes específicas, as classes derivadas agem como versões melhoradas para problemas específicos.
    • Ocultação de Informações.

    Polimorfismo¶

    Em programação, Polimorfismo se refere ao mesmo objeto exibindo diferentes formas e comportamentos.

    Polimorfismo serve para diminuir o trabalho do desenvolvedor, quando for necessário criar subclasses mais específicas com comportamentos e atributos únicos o desenvolvedor pode alterar o código em áreas específicas onde as respostas devem ser diferentes. Todos as outras partes do código permanecem intactas.

    Implementando Polimorfismo com Herança¶

    Ao implementar uma classe derivada de uma classe base, ela herda todos os métodos, porém algum método específico pode ser reescrito para substituir o comportamento desejado.

    Sobrecarregamento de Operadores¶

    Operadores em Python podem ser sobrescritos para funcionar de forma definida pelo usuário. Sempre que um operador é chamado, uma função específica do python é chamada, por exemplo:

    + invoca o método __add__.

    O comportamento de cada operador pode ser definido para uma classe específica ao definir o método especial para ele.

    Operador Método
    + __add__(self, other)
    - __sub__ (self, other)
    / __truediv__ (self, other)
    * __mul__ (self, other)
    < __lt__ (self, other)
    > __gt__ (self, other)
    == __eq__ (self, other)

    Classes Base Abstratas (ABC)¶

    Classes Base Abstratas definem um conjunto de métodos e propriedades que uma classe deve implementar a fim de ser considerada uma instância duck-type daquela classe.

    Para definir uma Classe Base Abstrata usamos o módulo abc, a classe base abstrata deve herdar a classe ABC embutida do python. Deve ser usado o decorador @abstractmethod acima do método que queremos declarar como um método abstrato.

    Classe pai: Herda a classe ABC e implementa métodos abstratos.

    Classe filha: Deve implementar os métodos abstratos obrigatoriamente.
