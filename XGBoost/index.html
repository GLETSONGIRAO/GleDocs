    Gradient Boosting com XGBoost¶

    XGBoost é uma biblioteca para Gradient Boosted Decision Trees altamente eficiente utilizada para classificação e regressão.

    XGBoost implemente uma versão altamente eficiente de Gradient Boosted Trees. Os modelos do XGBoost treinam muito mais rápido do que os modelos em scikit-learn.

    Em competições de ciência de dados para pequenos e médios conjuntos de dados, XGBoost está sempre os modelos com melhores resultados.

    Estruturas de dados Básicas¶

    A estrutura de dados básica para XGBoost é a DMatrix, que representa uma matriz de dados. A DMatrix pode ser construída a partir de arrays NumPy.

    O código abaixo criar objetos DMatrix com e sem rótulos.

    import xgboost as xgb
dmat1 = xgb.DMatrix(data) # data = np.array
# com rótulos
labels = np.array([0, 1])
dmat2 = xgb.DMatrix(data, label=labels)

    O objeto DMatrix pode então ser usado para treinar e usar um objeto Booster, que representa a Gradient Boosted Decision Tree.

    A função train do XGBoost nos permite treinar um Booster com um conjunto de parâmetros especificado.

    dtrain = xgb.DMatrix(data, label=labels)

params = {
  'max_depth': 0,
  'objective': 'binary:logistic',
  'eval_metric':'logloss'
}
bst = xgb.train(params, dtrain)
    Lista de possíveis parâmetros para Tree Booster:
    • eta [default=0.3, alias: learning_rate]
      • gamma [default=0, alias: min_split_loss]
      • max_depth [default=6]
      • min_child_weight [default=1]
      • max_delta_step [default=0]
      • subsample [default=1]
      • sampling_method [default= uniform]
    • uniform

    • gradient_based

    • colsample_bytree, colsample_bylevel, colsample_bynode [default=1]

    • lambda
    • alpha
    • tree_method
    • sketch_eps [default=0.03]
    • scale_pos_weight [default=1]
    • updater
    • refresh_leaf [default=1]
    • process_type [default= default]
    • grow_policy [default= depthwise]
    • depthwise
    • lossguide
    • max_leaves [default=0]
    • max_bin, [default=256]
    • predictor, [default= auto]
    • auto
    • cpu_predictor
    • gpu_predictor
    • num_parallel_tree, [default=1]
    • monotone_constraints
    • interaction_constraints

    Usando um Booster¶

    Depois de treinar o Booster podemos avaliar e usar para fazer predições.

    deval = xgb.DMatrix(eval_data, label=eval_labels)
dpred = xgb.DMatrix(new_data)
predictions = bst.predict(dpred)

    Validação Cruzada¶

    Escolhendo parâmetros¶

    Como existem muitos parâmetros no xgb e muitos valores possíveis para cada parâmetro, geralmente é necessário tunar e para isso utilizar Validação Cruzada.

    A função cv executa validação cruzada para um conjunto de parâmetros em um conjunto de dados.

    cv_results = xgb.cv(params, dtrain)

    A saída de cv é um DataFrame (pandas) que contém os resultados de treino e teste para média e desvio padrão de cada iteração. O nfold padrão é 3 e o número de iterações num_boost_round padrão é 10.

    cv_results = xgb.cv(params, dtrain, num_boost_round=5)

    Guardando Boosters¶

    Depois de achar os melhores parâmetros para um Booster podemos salvar o modelo é um arquivo binário. Cada Booster contém uma função chamada save_model, que salva o arquivo binário do modelo em um arquivo.

    bst = xgb.train(params, dtrain)
bst.save_model('model.bin')

    Podemos restaurar um Booster a partir de um arquivo binário com a função load_model, mas primeiro precisamos inicializar um Booster vazio.

    new_bst = xgb.Booster()
new_bst.load_model('model.bin')

    XGBoost Classifier¶

    Podemos utilizar XGBoost para classificação e regressão seguindo uma API com o mesmo estilo do Scikit Learn.

    Para classificação o modelo da API é chamado XGBClassifier e pode ser treinado com o método fit e arrays NumPy.

    model = xgb.XGBClassifier(use_label_encoder=False, eval_metric='logloss')
model.fit(data, labels)
predictions = model.predict(new_data)

    A função predict do XGBClassifier retorna a predição real e não as probabilidades.

    XGBoost Regressor¶

    Objeto XGBRegressor.

    model = xgb.XGBRegressor(max_depth=2)
model.fit(data, labels)

    Feature Importance¶

    Depois do treinamento, podemos ver a importância dos atributos de cada conjunto de dados utilizado a propriedade do modelo feature_importances_.

    model = xgb.XGBClassifier(objective='multi:softmax', eval_metric='mlogloss', use_label_encoder=False)
model.fit(data, labels)
print(model.feature_importances_)

    Plotando as importâncias com a função plot_importance:

    model = xgb.XGBRegressor()
model.fit(data, labels)

xgb.plot_importance(model)
plt.show()

    O resultado é um gráfico de barras dos F-scores para cada atributo.

    Por padrão, a função usa feature weight como métrica de importância, que diz o quão frequente o atributo aparece na Boosted Decision Tree.

    Podemos selecionar uma métrica diferente com o argumento importance_type:

    xgb.plot_importance(model, importance_type='gain')

    Tunando Hiperparâmetros¶

    Usando a API para estilo sklearn podemos usar os modelos com a API real do sklearn.

    Usando GridSearchCV para tunar hiperparâmetros:

    model = xgb.XGBClassifier(objective='binary:logistic', eval_metric='logloss', use_label_encoder=False)
params = {'max_depth': range(2, 5)}

from sklearn.model_selection import GridSearchCV
cv_model = GridSearchCV(model, params, cv=4)

cv_model.fit(data, labels)
  cv_model.best_params_['max_depth']

    Parâmetros comumente tunados são max_depth e eta.
