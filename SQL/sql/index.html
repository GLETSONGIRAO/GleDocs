    Introdução ao SQL (MySQL)¶

    Fundamentos de SQL¶

    Criando um Banco de dados¶

    Um banco de dados é um container que contém todas as tabelas. Uma tabela é um container para um subconjunto de dados. Uma tabela contém dados organizados em linhas e colunas. Uma coluna é uma parte dos dados que é um atributos de uma entidade. Uma linha é um conjunto de colunas que definem os atributos de uma entidade.

    Sintaxe

    CREATE DATABASE MovieIndustry;

    CREATE DATABASE IF NOT EXISTS 
MovieIndustry;

    SHOW DATABASES irá listar todas as databases presentes no sistema.

    Podemos excluir uma database com o comando DROP.

    DROP DATABASE MovieIndustry;

    Tipos de dados¶

    As principais categorias de tipos de dados são:

    • Numéricos: INT, BIGINT, TINYINT, DECIMAL, etc.
    • Tempo e Data: DATE, TIME, TIMESTAMP, YEAR, etc.
    • String: VARCHAR, CHAR, ENUM, SET, BLOB, etc.
    • JSON: JSON.
    • Dados Espaciais: Representam localização, tamanho e forma de um objeto no planeta Terra. MySQL suporta tipos de dados espaciais como GEOMETRY, POINT, etc.

    Criando a Tabela Actors para MovieIndustry:

    Nome da Coluna Tipo da Coluna
    FirstName VARCHAR(20)
    SecondName VARCHAR(20)
    DoB DATE
    Gender ENUM('Male', 'Female', 'Other')
    MaritalStatus ENUM(‘Married’,‘Divorced’,‘Single’)
    NetWorthInMilions DECIMAL

    Criando uma Tabela¶

    Sintaxe

    CREATE TABLE tableName (s
col1 <dataType> <Restrictions>,
col2 <dataType> <Restrictions>,
col3 <dataType> <Restrictions
<Primary Key or Index definitions>);

    Criando a tabela Actors:

    CREATE TABLE Actors (
FirstName VARCHAR(20),
SecondName VARCHAR(20),
DoB DATE,
Gender ENUM('Male','Female','Other'),
MaritalStatus ENUM('Married', 'Divorced', 'Single'),
NetWorthInMillions DECIMAL );

    SHOW TABLES; mostra que a tabela foi criada.

    DESC Actors; mostra as informações da tabela.

    Sintaxe completa para definir uma coluna:

    columnName columnType [NOT NULL | NULL] [DEFAULT columnValue]

    Criando tabela com restrições¶

    CREATE TABLE Actors (
Id INT AUTO_INCREMENT,
FirstName VARCHAR(20) NOT NULL,
SecondName VARCHAR(20) NOT NULL,
DoB DATE NOT NULL,
Gender ENUM('Male','Female','Other') NOT NULL,
MaritalStatus ENUM('Married', 'Divorced', 'Single', 'Unknown') DEFAULT "Unknown",
NetWorthInMillions DECIMAL NOT NULL,
PRIMARY KEY (Id));

    A coluna ID é a chave primária que identifica unicamente cada ator, e está atribuída com a feature AUTO_INCREMENT do MySQL, onde a primeira linha inicia em 1 e cada linha posterior será incrementada em 1.

    Criando Tabela Temporária¶

    Com MySQL é possível criar tabelas temporárias. Tabelas temporárias são persistidas pela duração da sessão e são removidas ao fim da sessão. Não é necessário que o usuário apague a tabela.

    Sintaxe: CREATE TEMPORARY TABLE tableName (...)

    Collation e Character Set¶

    Uma database pode conter caracteres de diversos idioma.

    Os conjuntos de caracteres disponíveis no servidor pode ser listado com o comando: SHOW CHARACTER SET;

    Também podemos listar Collations: SHOW COLLATION;

    Inserindo Dados¶

    Podemos inserir dados em uma tabela usando o comando INSERT:

    INSERT INTO table (col1, col2 … coln)
VALUES (val1, val2, … valn);

    Inserindo uma linha:

    INSERT INTO Actors ( 
FirstName, SecondName, DoB, Gender, MaritalStatus, NetworthInMillions) 
VALUES ("Brad", "Pitt", "1963-12-18", "Male", "Single", 240.00);

    Inserindo Múltiplas Linhas:

    INSERT INTO Actors ( 
FirstName, SecondName, DoB, Gender, MaritalStatus, NetworthInMillions) 
VALUES 
("Jennifer", "Aniston", "1969-11-02", "Female", "Single", 240.00),
("Angelina", "Jolie", "1975-06-04", "Female", "Single", 100.00),
("Johnny", "Depp", "1963-06-09", "Male", "Single", 200.00);

    Também podemos pular o nome das colunas e ir diretamente para os valores:

    INSERT INTO Actors 
VALUES (DEFAULT, "Dream", "Actress", "9999-01-01", "Female", "Single", 000.00);

    Inserir Linha com todos os valores nulos:

    INSERT INTO Actors () VALUES ();

    Usando a sintaxe de SET para agrupar pares Coluna e Valor:

    INSERT INTO Actors SET DoB="1950-12-12", FirstName="Rajnikanth", SecondName="",  Gender="Male", NetWorthInMillions=50,  MaritalStatus="Married";

    Querying¶

    Usamos o comando SELECT.

    SELECT col1, col2, … coln
FROM table
WHERE <condition>

    Selecionando todas as linhas de colunas: *

    SELECT * from Actors;

    Selecionando colunas: SELECT <columns> FROM <TableName>

    SELECT FirstName, SecondName from Actors;

    Filtrando linhas de acordo com condições: cláusula WHERE:

    SELECT FirstName, SecondName from Actors WHERE NetWorthInMillions > 500;

    Operadores que podem ser usados com WHERE¶

    Operator Propósito
    > Greater than operator
    >= Greater than or equal to operator
    < Less than operator
    <= Less than or equal to operator
    != Not equal operator
    <> Not equal operator
    <=> NULL-safe equal to operator
    = Equal to operator
    BETWEEN … AND … Whether a value is within a range of values
    COALESCE() Return the first non-NULL argument
    GREATEST() Return the largest argument
    IN Whether a value is within a set of values
    INTERVAL Return the index of the argument that is less than the first argument
    IS Test a value against a boolean
    IS NOT Test a value against a boolean
    IS NOT NULL NOT NULL value test
    IS NULL NULL value test
    ISNULL() Test whether the argument is NULL
    LEAST() Return the smallest argument
    LIKE Simple pattern matching
    NOT BETWEEN … AND … Whether a value is not within a range of values
    NOT IN() Whether a value is not within a set of values
    NOT LIKE Negation of simple pattern matching
    STRCMP() Compare two strings

    Operador Like¶

    Um dos operadores que podem ser usados com a cláusula WHERE. O operador LIKE nos permite recuperar linhas em baseado em uma correspondência de padrão para alguma coluna.

    Sintaxe:

    SELECT col1, col2, … coln
FROM table
WHERE col3 LIKE "%string%"

    Exemplo:

    SELECT * from Actors WHERE FirstName LIKE "Jen%";
    • O símbolo % é um wildcard que corresponde a 0 ou mais caracteres.
    • _ corresponde a exatamente 1 caractere.

    A cláusula LIKE também pode ser usadas com comandos SQL diferentes de SELECT.

    SHOW DATABASES LIKE "M%";

    Combinando Condições¶

    Podem ser combinadas múltiplas condições com WHERE usando operadores como AND e OR.

    SELECT col1, col2, … coln
FROM table
WHERE col3 LIKE "%some-string%"
AND
col4 = 55;

    Exemplo:

    SELECT * FROM Actors WHERE FirstName > "B" AND NetWorthInMillions > 200;

    Combinando Operador AND e operador OR:

    SELECT * FROM Actors WHERE (FirstName > 'B' AND FirstName < 'J') OR (SecondName >'I' AND SecondName < 'K');

    ORDER BY¶

    A cláusula ORDER BY ordena as linhas da consulta, em ordem decrescente DESC ou crescente ASC.

    Sintaxe:

    SELECT col1, col2, … coln
FROM table
WHERE col3 LIKE "%some-string%"
ORDER BY col3

    Exemplo:

    SELECT * FROM Actors ORDER BY NetWorthInMillions DESC, FirstName DESC;

    LIMIT¶

    Tabelas no mundo real frequentemente têm milhões de linhas. Consultas podem retornar muitas linhas, o que pode ser problemático para o desempenho do banco de dados. A cláusula LIMIT nos permite restringir o número de linhas retornadas em uma consulta.

    Sintaxe

    SELECT col1, col2, … coln
FROM table
WHERE col3 LIKE "%some-string%"
ORDER BY col3
LIMIT 10;

    Exemplo:

    SELECT FirstName, SecondName from Actors ORDER BY NetWorthInMillions DESC LIMIT 3;

    A palavra chave OFFSET retorna a quantidade limite de linhas que estão situadas após a quantidade determinada pelo OFFSET.

    SELECT FirstName, SecondName from Actors ORDER BY NetWorthInMillions DESC LIMIT 4 OFFSET 3;
# ou
SELECT FirstName, SecondName from Actors ORDER BY NetWorthInMillions DESC LIMIT 3,4;

    LIMIT <offset>, <number_of_row_to_print>;

    Deletando Dados¶

    DELETE pode ser usado para deletar dados de uma tabela de acordo com uma consulta específica.

    Sintaxe

    DELETE FROM table
WHERE col3 > 5
ORDER BY col1
LIMIT 5;

    Para deletar todas as linhas de uma tabela pode ser usado TRUNCATE.

    TRUNCATE table;

    UPDATE¶

    UPDATE pode ser usado para mudar os valores de uma ou mais linhas e colunas.

    Sintaxe

    UPDATE table
SET col1 = val1, col2 = val2, … coln = valn
WHERE <condition>
ORDER BY col5
LIMIT 5;

    Chave Primária e Índices¶

    Sempre que uma consulta que contém a cláusula WHERE feita, o MySQL tem que fazer um escaneamento completo da tabela para achar as linhas correspondente. Uma escaneamento linear de uma tabela não é muito eficiente.

    Adicionar um índice para a tabela pode aumentar a velocidade da procura significativamente. O propósito principal de um índice é fornecer uma representação ordenada dos dados indexados.

    Índice Clusterizado¶

    Quando inserimos linhas em uma tabela, elas são escritas no mídia de armazeno físico, tal como no HD. No caso de índice clusterizado, as linhas são ordenadas e mantidas em uma estrutura de B-tree (ou R-Tree no caso de índice espacial).

    O arranjo físico das linhas no disco mimetiza a ordem lógica definida pelo índice. Não existe estrutura de dados separada que mantém as linhas da tabela. É uma forma de informar ao sistema de banco de dados para agrupar valores fisicamente próximos uns aos outros no disco que estão próximos no índice.

    B-trees e a versão modificada B+ trees, são estruturas de dados comuns usadas para armazenar dados em discos. Tabelas muito grandes não cabem na memória principal do computador e apenas porções de tais tabelas são trazidas do disco rígido para a memória principal. A procura pelos dados requisitados se torna mais fácil com B-trees pois elas garantem um número fixo de leituras, já que B-trees são balanceadas.

    Em B+ trees, os dados são armazenados em nós folha. A raiz e os nós internos contém a chave pela qual a B+ tree é ordenada.

    MySQL armazena dados em uma entidade chamada page. Uma page é a menor unidade de dados que uma database pode escrever ou ou ler de um disco. Uma page contém linhas e forma o nó folha da B+ tree.

    Em MySQL o tamanho padrão de uma page é 16kb, porém é configurável.

    Alterações¶

    Podemos alterar entidades: renomear tabelas, adicionar, remover ou renomear colunas, mudar o tipo de uma coluna existente, etc.

    Sintaxe

    ALTER TABLE table
CHANGE oldColumnName newColumnName <datatype> <restrictions>;

    ALTER TABLE Actors CHANGE FirstName First_Name varchar(120);

    Podemos usar MODIFY para alterar o tipo ou as cláusulas de uma coluna.

    ALTER TABLE Actors MODIFY First_Name varchar(20) DEFAULT "Anonymous";

    Também CHANGE, mas o nome da coluna deve ser repetido duas vezes se não for renomear a coluna.

    ALTER TABLE Actors CHANGE First_Name First_Name varchar(20) DEFAULT "Anonymous";

    ADD para adicionar uma coluna. AFTER ou FIRST para escolher a posição.

    ALTER TABLE Actors ADD MiddleName varchar(100) FIRST;

    Adicionar índices com ADD INDEX:

    ALTER TABLE table
ADD INDEX indexName (col1, col2, … coln);

    # Limitando o índice em 10 caracteres
ALTER TABLE Actors ADD INDEX nameIndexWithOnlyTenChars (FirstName(10));

    Alias¶

    Alies é um "apelido", um nome temporário que substitui o nome de uma tabela ou coluna para ajudar a escrever consultas mais legíveis. Podemos usar alias com colunas, tabelas e funções MySQL.

    Sintaxe

    SELECT col1
AS aliasCol1
FROM table;

    Exemplo Funções

    SELECT CONCAT(FirstName,' ', SecondName) AS FullName FROM Actors;

    Exemplo Tabela

    SELECT FirstName FROM Actors AS tbl WHERE tbl.FirstName='Brad' AND tbl.NetWorthInMillions > 200;

    SELECT t1.FirstName, t1.NetworthInMillions
FROM Actors AS t1,
Actors AS t2
WHERE t1.NetworthInMillions = t2.NetworthInMillions
AND t1.Id != t2.Id;

    DISTINCT¶

    Retorna as linhas únicas em uma consulta.

    DISTINCT é um filtro pós processamento, é aplicado após o resultado de uma consulta.
