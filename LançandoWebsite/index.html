    Publicando um Website

    Introdução¶

    Localhost¶

    O termo "localhost" se refere ao endereço de IP padrão do computador. Localhost (Loopback IP Adress)

    É geralmente usado para hospedar um website no endereço 127.0.0.1 que é acessível apenas no seu sistema. Ele nos permite testar e otimizar o website até que todos os bugs estejam consertados, o que nos dá o benefício de pagar apenas quando o site estiver 100% pronto para ser hospedado em um servidor web.

    Métodos para construir um Website¶

    Criar o site do zero em um localhost usando qualquer tecnologia de desenvolvimento web e então:

    1. Comprar um domínio e um serviço de hospedagem na web separadamente.
    2. Comprar um domínio e um serviço de hospedagem na web separadamente.
    3. Comprar tudo em um pacote configurando o Website usando um criador de websites.

    Três elementos são necessários para colocar um site online:

    • Domínio
    • DNS
    • Serviços de Hospedagem Web

    Passos para lançar um Website:

    • Registrar um Nome de Domínio
    • Achar um serviço de hospedagem
    • Publicar o website
    • Teste e Manutenção
    • SEO e Web Analytics

    Nome do Domínio¶

    O nome do domínio é um endereço único (URL) que o usuário deverá digitar para visitar o site. Nomes de domínio são utilizados no lugar de endereços de IP porque são mais fáceis de lembrar do que um número de IP.

    Um URL consiste basicamente de 4 partes:

    • Domínio de Segundo Nível: Usualmente o nome verdadeiro do website.
    • Domínio de Topo: A última parte que segue após o nome
    • Subdomínio: A parte que vem antes do domínio de segundo nível
    • Protocolo: HTTP ou HTTPS

    DNS¶

    DNS (Domain Name Server) mapeia a URL para o real endereço de IP. Ele é formado por um cojunto de registro de configurações , cada registro aponta para uma hospedagem na web.

    Elementos do DNS:

    • Um registro: Este registro aponto para o endereço de IP do website. Uma vez queo domínio está apontando para o registro correto então podemos ser direcionados para a hospedagem web correta.
    • Registro mx: Este registro aponta os emails que vão para o seu dome de domínio para o serviço de hospedagem e de email.

    Hospedagem Web¶

    Os arquivos do website são armazenados no servidor de hospedagem.

    Existe uma grande variedade de serviços de hospedagem web disponíveis.

    Quando acessamos uma URL, a requisição vai para o servidor DNS que retorna o endereço de IP, essa requisição então é feita para o servidor de hospedagem web nesse endereço e então o servidor manda a resposta.

    Registrador de Nome de Domínio¶

    É preciso de um registrador de nome de domínio para comprar um domínio para um website. Todos os registradores são aprovados pelo ICANN (a organização sem fins lucrativos que administra o sistema de nome de domínio).

    A maioria dos Registradores tem um conjunto similar de características e também desvantagens. Existem centenas de Registradores disponíveis na internet hoje em dia, os mais populares são:

    • GoDaddy
    • Hover
    • Domain.com
    • Bluehost
    • Hostgator
    • Namecheap

    Como Escolher?¶

    Preço¶

    Os valores geralmente são cobrados anualmente ou até em intervalos de mais de um ano.

    Se não houver renovação automática, corre o risco de alguém comprar o domínio.

    Suporte ao Consumidor¶

    Alguns registradores podem ser baratos mas oferecer um péssimo suporte, o que causará problemas a longo prazo.

    Features adicionais¶

    • Se seguram domínio após expiração.
    • Se permitem compra de certificados.
    • Se permitem transferência de domínio.

    Passos para Registrar um nome de domínio.¶

    Escolher um nome de domínio¶

    Sufixo Uso
    .com Uso comercial
    .biz Para uso em negócios
    .net Alternativa para .com
    .org Para organizações sem fins lucrativos
    .edu Para institutos educacionais

    Também existem Código de país em adição a esses domínios, como: .br, .uk, .de.

    Checar a disponibilidade do nome¶

    Usar um motor de busca como DomainTools.

    Registrar o nome do domínio com um registrador de domínios¶

    As taxas para registrar e renovar o domínio anualmente são por volta de 10 a 15 dólares.

    Serviço de Hospedagem¶

    Um serviço de hospedagem armazena o site em um servidor e o publica online para acesso público.

    Podemos hospedar um site por conta própria, mas precisaríamos de um servidor web poderoso para lidar com o tráfego de usuários.

    Tipos de Hospedagem¶

    Servidores Dedicados¶

    Um servidor dedicado aloca todos os seus recursos para hospedar apenas nosso website. Nunca precisamos compartilhar o poder de computação do servidor com mais alguém. Podendo até ser usado para gerenciar múltiplos sites e softwares.

    Este é o tipo mais caro de servidor, com valores mensais variando de $50 a $100.

    Bluehost and HostGator são serviços de hospedagem populares.

    Hospedagem Gerenciada¶

    Hospedagem gerenciada é uma subcategoria de servidores dedicados onde o provedor do serviço administra o servidor para nós. O que garante uma maior segurança e reduz a chances de falha no servidor. Mais caro.

    Servidor Privado Virtual (VPS)¶

    Um servidor que roda vários servidores virtuais, cada um dedicado a um website. Ainda temos a flexibilidade de usar o servidor em um ambiente privado, mas a performance não será tão boa quando em um servidor puramente dedicado.

    Um VPS típico custa por volta de $30 por mês. Namecheap e InMotion Hosting são serviços comuns.

    Hospedagem Compartilhada¶

    Não tão eficiente, porque os recursos são compartilhados, tráfego em um site afeta os outros.

    Custa menos de $10 por mês.

    Hospedagem em Nuvem¶

    O desenvolvimento mais atual no mundo das hospedagens de websites. Os recursos para o website estão divididos entre múltiplos servidores virtuais que são parte da nuvem. Podemos adquirir quantos servidores virtuais na nuvem quisermos e podemos ainda especificar os recursos desejados (RAM, armazenamento...).

    Mais protegidos contra falha do servidor, os dados podem ser movidos entre qualquer dos servidores ao redor do mundo.

    Em relação aos custos, pagamos apenas pelo que usarmos, os custos se ajustam automaticamente à escala de uso.

    Principais provedores deste serviço: Amazon, Google e Microsoft.

    Utilidades Necessárias¶

    O servidores que escolhermos deve ter as seguintes ferramentas acessíveis:

    • PHP, Perl, JavaScript (web development languages)
    • File Transfer Protocol (file accessibility)
    • .htaccess (Apache server configuration)
    • Server Side Includes
    • MySQL or MongoDB (database management)
    • SSH (remote server communication)
    • Cron (job scheduler)
